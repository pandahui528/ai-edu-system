# AI 原生教育创新项目快速落地系统（Global Spec v1.1）

## 0. 北极星目标
以“一人公司”模式，在教育领域持续验证真实市场需求：每月交付至少 1 个可被真实用户使用的 AI 原生产品原型，并通过多智能体协作 + 全局复盘，让系统持续进化。

## 1. 四条不可违背原则
1) 真实：必须真实用户可用、可反馈  
2) 快速：7–10 天出可用原型；先跑通再优化  
3) 人在回路：仅在“业务方向/产品战略/成本支出”强制确认  
4) 可复用优先：资产沉淀为模板、契约、playbook；反复复用

## 2. 组织结构（精简 6 个智能体）
- 01 产品与决策 Agent（唯一允许打断 CEO）
- 02 市场与验证 Agent
- 03 产品实现 Agent（MVP 裁剪 + 用户路径）
- 04 技术执行 Agent（代码/部署/接口）
- 05 增长与分发 Agent（微信触达/反馈）
- 06 全局复盘与进化 Agent（外挂，不参与日常）

## 3. 协作流水线（固定顺序）
市场与验证 → 产品实现 → 技术执行 → 增长与分发 → 产品与决策 → CEO 一句话决策

任何 Agent 不得跳过顺序或并行抢决策权。

## 4. 原型最低合格标准
- 无学习成本；30 秒内完成首次体验
- ≤ 3 步完成一次完整使用
- 自动收集最小反馈（打开/完成/失败原因/一句话建议）
- 所有接口返回必须带 traceId 便于定位

## 5. 决策点（仅三类强制确认）
1) 业务方向：目标人群/场景/价值主张发生变化  
2) 产品战略：MVP范围升级、转向、停止、定价策略  
3) 成本支出：新增费用或超预算行为

其余执行动作默认授权智能体自主推进。

## 6. 系统进化机制
- 每个项目结束必须复盘
- 复盘结论必须回流更新：templates/、playbooks/、agents/契约或提示词
- 系统任何改动的“唯一入口”是复盘结论（避免凭感觉频繁改系统）
