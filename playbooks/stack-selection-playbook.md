# 技术栈选择与组合 Playbook（按项目特征选型）

## 目标
按项目特征选择“最快可用”的组合，避免接口配置/环境变量/上传/域名/CORS耗时。

## 选型输入（必须收集）
- 形态：小程序原生 / H5+小程序承载
- 输入：文本/图片/音频/视频；最大文件体积
- 时延：即时(<10s)/可等待/异步
- 登录：无/微信登录
- 合规：数据是否必须国内；是否可第三方

## 默认优先策略（节省时间）
1) 先选 H5+小程序承载（最快验证），除非必须原生能力  
2) 上传类：优先 COS 直传（避免网关/云函数限制）  
3) 后端：简单同步优先云函数；长耗时转 CloudBase Run 或异步队列  
4) LLM：优先国内可用且稳定的供应商；必须提供“回退模型/降级输出”
